# 🚀 คู่มือการติดตั้ง Telegram-First Standalone Maintenance System

ระบบบำรุงรักษาผ่าน Telegram พร้อมหน้า Dashboard และ AI Integration (Gemini 2.0 Flash)

---

## 📌 ขั้นตอนที่ 1: การตั้งค่าฝั่ง Telegram Bot

1. **สร้างบอทใหม่:**
   - เปิดแอป Telegram แล้วค้นหา `@BotFather`
   - พิมพ์คำสั่ง `/newbot` แล้วทำตามขั้นตอนเพื่อตั้งชื่อบอทและ Username (ต้องลงท้ายด้วย bot)
   - เมื่อเสร็จสิ้น คุณจะได้รับ **Token** (เช่น `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`) ให้คัดลอกเก็บไว้

2. **การตั้งค่า Group Chat (สำหรับกระดานประกาศแจ้งงานใหม่):**
   - **แนวคิดการทำงาน:** ระบบนี้ใช้การสั่งงานแบบ 1:1 (ทักแชตส่วนตัว) เป็นหลัก เพื่อความปลอดภัย แต่จะใช้ Group Chat เป็น "กระดานประกาศแจ้งเตือนช่างทุกคน" เมื่อมีงานใหม่เข้ามา
   - สร้าง Group ใน Telegram (เช่นตั้งชื่อว่า "ทีมซ่อมบำรุง") และดึงบอทของคุณเข้าไปในกลุ่ม
   - เพิ่มบอทเครื่องมืออย่าง `@RawDataBot` หรือ `@userinfobot` เข้าไปในกลุ่มเพื่อดูรหัส Chat ID ของกลุ่มนั้น (รหัสกลุ่มมักจะนำหน้าด้วยเครื่องหมายลบ เช่น `-100123456789`)
   - นำรหัส Chat ID ที่ได้ไปใส่ไว้ในตัวแปร `TELEGRAM_GROUP_CHAT_ID` ในไฟล์ `Config.js`

3. **การหา Chat ID ของผู้บริหาร (สำหรับสิทธิ์ VIEW_REPORT และ CREATE_JOB):**
   - ให้ผู้บริหารทักแชตส่วนตัวไปหาบอทของคุณ
   - เช็ค Chat ID ของผู้บริหารผ่านการเรียก API `https://api.telegram.org/bot<TOKEN>/getUpdates` บนเบราว์เซอร์
   - นำตัวเลข Chat ID (เช่น `987654321`) เก็บไว้เป็น `BOSS_TELEGRAM_CHAT_ID`

## 📌 ขั้นตอนที่ 2: การตั้งค่าฝั่ง Google Apps Script (GAS) และ Spreadsheet

1. **สร้างฐานข้อมูล (Google Sheets):**
   - สร้าง Google Sheets ใหม่ 1 ไฟล์
   - สร้างชีตจำนวน 3 ชีต ได้แก่ `Jobs`, `Technicians`, `Admins`
   - หัวคอลัมน์ `Jobs`: Job_ID, Timestamp, Reporter_Name, Reporter_Phone, Location, Issue_Detail, Category, Technician_ID, Technician_Name, Status, Photo_URL, Closing_Note
   - หัวคอลัมน์ `Technicians`: Tech_ID, Name, Google_Email, Telegram_Chat_ID, Status
   - หัวคอลัมน์ `Admins`: Admin_ID, Name, Google_Email
   - คัดลอก ID ของ Spreadsheet จาก URL (ส่วนที่อยู่ระหว่าง `/d/` และ `/edit`)

2. **สร้างพื้นที่จัดเก็บรูปภาพและรายงาน (Google Drive):**
   - สร้างโฟลเดอร์ใหม่ใน Google Drive จำนวน 2 โฟลเดอร์ (สำหรับเก็บรูปภาพหน้างาน และ สำหรับเก็บไฟล์ PDF รายงาน)
   - คัดลอก ID โฟลเดอร์จาก URL เก็บไว้เป็น `GOOGLE_DRIVE_FOLDER_ID` (เก็บรูป) และ `GOOGLE_DRIVE_PDF_FOLDER_ID` (เก็บ PDF)

3. **เตรียมโค้ด GAS:**
   - ในหน้า Google Sheets ไปที่เมนู `ส่วนขยาย (Extensions)` > `Apps Script`
   - คัดลอกโค้ดจากไฟล์ `backend/Config.js` และ `backend/Code.js` ไปวางในโปรเจกต์
   - แก้ไขตัวแปรใน `Config.js` ให้ตรงกับข้อมูลที่เตรียมไว้ (ใส่ Token, ID ต่างๆ และ Gemini API Key)

4. **เปิดใช้งาน Google Drive API ใน GAS:**
   - ในหน้า Apps Script เมนูด้านซ้าย เลือก **"บริการ" (Services)** (รูปเครื่องหมาย +)
   - ค้นหาและเลือก **Google Drive API** แล้วกดปุ่มเพิ่ม (Add) เพื่อให้บอทสามารถเซฟรูปได้

5. **Deploy เป็น Web App:**
   - กดปุ่ม **การทำให้ใช้งานได้ (Deploy)** มุมขวาบน > เลือก **การทำให้ใช้งานได้รายการใหม่ (New deployment)**
   - เลือกประเภทเป็น **เว็บแอป (Web app)**
   - ระบุผู้มีสิทธิ์เข้าถึง (Who has access) เป็น **ทุกคน (Anyone)**
   - กด Deploy แล้วกดยอมรับสิทธิ์การเข้าถึงข้อมูล (Authorize Access)
   - คัดลอก **Web App URL** ที่ได้ (ขึ้นต้นด้วย `https://script.google.com/...`) เก็บไว้

## 📌 ขั้นตอนที่ 3: การตั้งค่า Webhook ของ Telegram

ให้เปิดเว็บเบราว์เซอร์แล้วนำ URL นี้ไปวาง โดยเปลี่ยน `<TOKEN>` และ `<WEB_APP_URL>` เป็นข้อมูลของคุณ:

```text
https://api.telegram.org/bot<TOKEN>/setWebhook?url=<WEB_APP_URL>
```

- กด Enter หากสำเร็จหน้าเว็บจะแสดงข้อความว่า `{"ok":true,"result":true,"description":"Webhook was set"}`
- ตอนนี้บอทของคุณสามารถรับข้อความและส่งเข้า GAS ได้แล้ว

## 📌 ขั้นตอนที่ 4: การตั้งค่า Frontend (Dashboard) และ Google OAuth

1. **สร้าง OAuth Client ID สำหรับระบบ Login:**
   - ไปที่ [Google Cloud Console](https://console.cloud.google.com/)
   - สร้างโปรเจกต์ใหม่ > ไปที่เมนู **APIs & Services** > **Credentials**
   - กด **Create Credentials** เลือก **OAuth client ID**
   - เลือก Application type เป็น **Web application**
   - ในส่วน **Authorized JavaScript origins** ให้ใส่ URL ของเว็บที่คุณจะนำไปโฮสต์ (เช่น `http://localhost:3000` สำหรับทดสอบ หรือ `https://your-username.github.io` สำหรับ GitHub Pages)
   - กดสร้างแล้วคัดลอก **Client ID** (ลงท้ายด้วย `apps.googleusercontent.com`)

2. **ปรับแก้ไฟล์ Frontend:**
   - เปิดไฟล์ `frontend/index.html`
   - นำ **Client ID** ไปใส่ตรง `data-client_id="YOUR_GOOGLE_CLIENT_ID_HERE.apps.googleusercontent.com"`
   - นำ **Web App URL** จากขั้นตอนที่ 2.5 ไปใส่ที่ตัวแปร `const GAS_WEBHOOK_URL = "YOUR_GAS_WEBAPP_URL_HERE";`

3. **โฮสต์หน้าเว็บด้วย GitHub Pages:**
   - สร้าง Repository ใหม่ใน GitHub
   - อัปโหลดไฟล์ `frontend/index.html` ขึ้นไป
   - ไปที่ Settings ของ Repository > เมนู **Pages** ด้านซ้าย
   - ตรงช่อง Source เลือก Branch `main` หรือ `master` แล้วกด Save
   - รอสักครู่ คุณจะได้รับ URL หน้าเว็บ เช่น `https://your-username.github.io/your-repo/`
   - *(อย่าลืมนำ URL นี้ไปเพิ่มใน Authorized JavaScript origins ของ Google Cloud Console ด้วย)*

## 📌 ภาคผนวก: ตัวอย่างคำสั่งที่บอทรองรับ (แบ่งตาม Role)

ระบบใช้ AI (Gemini) ในการตีความเจตนา (Intent) ดังนั้นคุณ **ไม่จำเป็นต้องพิมพ์คำเป๊ะๆ** ตามนี้ สามารถพิมพ์ด้วยภาษาธรรมชาติทั่วไปได้เลยครับ:

### 👨‍🔧 สำหรับช่าง (Technician)
- **ลงทะเบียนเข้าใช้งาน (ครั้งแรก):** `/register PTTEP2026 สมชาย ใจดี`
- **ขอดูงานค้างของตัวเอง:** "วันนี้มีคิวซ่อมที่ไหนบ้าง", "ขอดูงานค้างหน่อย"
- **ส่งรูปปิดงาน (ต้องแนบรูปมาด้วยเสมอ):** *(แนบรูปภาพพร้อมแคปชัน)* "ปิดงาน JOB-123 ครับ ซ่อมแอร์เสร็จแล้ว เติมน้ำยาไป 10 ปอนด์"

### 👨‍💻 สำหรับแอดมิน (Admin)
- **เปิดงานซ่อมใหม่:** "แจ้งซ่อมแอร์น้ำหยดที่ห้องประชุมชั้น 3" 
  - 📸 **การเปิดงานพร้อมแนบรูป (Optional):** คุณสามารถ "ส่งรูปภาพ" พร้อมกับพิมพ์คำบรรยาย (Caption) ได้เลย
    - *แบบที่ 1 (ข้อมูลครบ):* ส่งรูปภาพ + พิมพ์แคปชัน "แอร์น้ำหยดที่ห้องประชุมชั้น 3" -> บอทจะเปิดงานและเซฟรูปรอซ่อมให้ทันที
    - *แบบที่ 2 (ข้อมูลไม่ครบ):* ส่งรูปภาพ + แคปชัน "เสียครับ" -> บอทจะเซฟรูปพักไว้ (Draft) และถามหาสถานที่ แอดมินแค่พิมพ์สถานที่ตอบกลับไป บอทก็จะเปิดงานพร้อมดึงรูปมาใส่ให้เลย
- **แจกจ่าย/โอนงาน:** "งาน JOB-123 ให้ช่างสมชายไปทำนะ", "เปลี่ยนคนรับผิดชอบ JOB-555 เป็นช่างสมรที"
- **ยกเลิกงาน (Cancel):** "ยกเลิกงาน JOB-123 เนื่องจากลูกค้าแจ้งซ่อมซ้ำ", "ขอยกเลิก JOB-999"
- **ตรวจสอบงานที่ยังไม่แจก:** "มีงานไหนยังไม่แจกช่างบ้าง", "งานที่ยังไม่มีคนทำ"
- **ตรวจสอบงานค้างทั้งหมด:** "ตอนนี้มีซ่อมอะไรค้างอยู่ในระบบบ้าง", "ขอดูงานค้างทั้งหมด"
- **เช็กภาระงานช่าง (Workload):** "เช็คโหลดงานของช่างแต่ละคนหน่อย", "ใครงานล้นมือบ้าง"
- **ดูรายละเอียดงาน:** "ขอดูรายละเอียดงาน JOB-999 หน่อย"

### 💼 สำหรับผู้บริหาร (Boss)
- **เรียกดูรายงานสรุป (PDF):** "ขอรายงานสรุปหน่อย", "อัปเดตงานทั้งหมดให้ดูที" *(บอทจะสรุปภาพรวม และสร้างไฟล์ PDF พร้อมดึงรูปหลักฐานมาแนบท้ายให้อัตโนมัติ)*
- **เรียกดูรายงานสถิติ (SLA):** "ขอรายงานสถิติ", "สรุป SLA การซ่อม", "สรุป top 10 คนซ่อมไว"
- *(สามารถสั่งงานเหมือนแอดมินได้ทุกคำสั่ง)*
